Parallel Computing

What if my computer does not allow this?

  • As an AMSE member you can fill a form on the intranet

  • Use GlobalProtect (VPN from AMU) to access the the VM outside AMSE

  • Follow the instructions - in Windows you click on Remote Desktop Connection

Principles

  • Divide your computation task into multiple tasks
  • Execute them simultaneously
  • Examples: life-cycle model of consumption-saving - see a practical guide to parallelization in Fern√°ndez-Villaverde and Valencia (2018), bootstrap‚Ä¶ + Less obvious: MCMC algorithms
  • Either define the tasks yourself:
    • joblib in Python
    • doParallel and foreach in R
    • parallel in Stata
  • ‚Ä¶ Or use options of the functions
    ‚áí No general rule, depends on your problem

Python - Parallel loops

from joblib import Parallel, delayed
import time

nb_cores = 5

start_time = time.time()
output = Parallel(n_jobs=nb_cores)(delayed(lambda t: time.sleep(1) or [f"{t}a", f"{t}b"]) (t) 
        for t in range(1, 16))
end_time = time.time()

print(f"Elapsed time: {round(end_time - start_time)} seconds")
print(output)

Python - Use options of a specific function

import time
import pandas as pd
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import GridSearchCV, train_test_split

n_jobs = 5

data = pd.read_csv("data.csv")
seed = 143
train, test = train_test_split(data, test_size=0.3, random_state=seed)

param_grid = {
    'n_estimators': [500],
    'max_features': [3, 15]
}

rf_model = RandomForestClassifier(random_state=seed)
grid_search = GridSearchCV(rf_model, param_grid=param_grid, cv=5, n_jobs=n_jobs)
start_time = time.time()
grid_search.fit(train.drop("class", axis=1), train["class"])
end_time = time.time()
print(f"Elapsed time: {round(end_time - start_time)} seconds")
print("Best parameters: ", grid_search.best_params_)
Elapsed time: 45 seconds

Causal Machine Learning

What is Causal Machine Learning?

  • Causality: what if individual i would have received treatment A instead of treatment B?

  • ML: individualized causal responses from observational or experimental data, high-dimensionality, variable selection

  • ML methods are not directly applicable:

    • Prediction, not causality
    • Naive plug-in strategies do not ensure common asymptotical guarantees
    • Many developments: causal forest (Athey, Tibshirani, and Wager 2019), double machine learning (Chernozhukov, Chetverikov, et al. 2018) and many others‚Ä¶

Illustration with my work

Policy Tree

A policy tree estimates which treatment arm to assign to whom. Its goal is to maximize welfare based on estimation of the potential outcomes associated to each treatment arm.

  • We wanted to implement a method described in Zhou, Athey, and Wager (2022)

  • In R, we can apply R code provided by the authors here

  • In Python, EconML proposes PolicyTree and DRPolicyTree

  • In practice: it was computing way faster
    ‚áí I was not sure whether it was implementing the same method, so even if we coded everything in Python, we chose to code in R for this part
